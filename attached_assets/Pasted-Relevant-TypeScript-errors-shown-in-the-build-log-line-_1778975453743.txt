Cause

TS2353 is the TypeScript "excess property check" error: an object literal includes property(ies) that are not allowed by the target type/interface. In short, code at those lines is creating or assigning an object literal whose keys do not match the expected type, and the TypeScript compiler is refusing to compile.

Open the three locations indicated and inspect the object literals:

Fix options (choose the one that best matches your intent):

Option A — Make the object match the declared type (recommended)

Change the object literal so it only contains properties defined on the target interface/type.
Example: if the expected type is NPCConfig { x:number; y:number }, change:
// before (extraProp not on type)
const cfg = { x: 0, y: 0, extraProp: 10 };
to
// after
const cfg = { x: 0, y: 0 };
// or, if extraProp is needed, add it to the type (see Option B)
Option B — Extend the type to include the property(ies)

If the extra properties are intended, add them to the target interface/type definition.
Example:
// add missing field to interface
interface NPCConfig {
  x: number;
  y: number;
  extraProp?: number; // <--- add this
}
Option C — Narrowly bypass the excess property check (use sparingly)

If you must keep the object literal as-is and cannot change the declared type, you can assert the type:
const cfg = { x: 0, y: 0, extraProp: 10 } as unknown as NPCConfig;
or assign via a temporary variable to avoid direct excess check:
const tmp = { x: 0, y: 0, extraProp: 10 };
const cfg: NPCConfig = tmp; // no excess check here
Note: this silences the type system and should be used only when you are sure the runtime shape is acceptable.
